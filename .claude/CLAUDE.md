# Claude Code 项目配置

## 使用限制

本 Claude Code 实例配置为**仅限在 xmf_alpha 账户下的 /home1/LT 目录中使用**。

### 配置说明

- **允许的用户**: xmf_alpha
- **允许的工作目录**: /home1/LT 及其子目录
- **项目路径**: /home1/LT/SGSAT-KT

### 安全限制

1. 不要在 /home1/LT 目录之外执行任何操作
2. 不要访问其他用户的文件或目录
3. 不要修改系统级别的配置文件
4. 仅限于 SGSAT-KT 项目的开发工作

### 工作目录检查

如果当前工作目录不在 /home1/LT 下，Claude Code 应拒绝执行操作。

## 开发规范

### 必读文件

每次执行新的任务前，**必须**先阅读以下内容，了解当前配置和项目全貌：

1. `README.md` — 项目结构和使用方法
2. `configs/default.yaml` — 所有训练参数、预设模式的唯一配置来源
3. `docs/` 目录下的文档 — 项目设计、训练指南、配置指南等

不得脱离这些文件自行假设参数或结构。

### 文档同步

每次对项目代码或配置进行修改后，**必须**同步更新所有相关文档，包括但不限于：

- `README.md` — 项目结构、使用方法变更
- `docs/` 目录下的对应文档 — 设计细节、参数说明变更
- `scripts/README.md` — 脚本使用说明变更

确保文档与代码始终一致，避免文档过时。

### 项目整洁

每次执行完命令后，**必须**检查项目中是否产生了垃圾文件或冗余信息，包括但不限于：

- 临时文件、缓存文件（`__pycache__`、`.pyc` 等）
- 未使用的配置文件或残留文件
- 重复或过时的文档内容

发现后及时清理，保持项目目录干净整洁。

### 文件管理约束

项目目录结构有明确分工，生成或修改文件时**必须**遵循以下规则：

| 目录 | 用途 | 规则 |
|------|------|------|
| `scripts/` | 可执行脚本（预计算、训练） | 禁止新增脚本，只修改现有 |
| `configs/` | 配置文件 | 训练参数统一在 `default.yaml` 管理 |
| `docs/` | 项目文档 | 修改代码后必须同步更新对应文档 |
| `DTransformer/` | 模型代码 | 禁止修改模型架构 |
| `utils/` | 工具函数 | 可新增工具模块 |
| `data/` | 数据文件 | 只读，不要修改或生成数据 |
| `output/` | 训练输出 | 自动生成，不要手动编辑 |
| `logs/` | 训练日志 | 自动生成，不要手动编辑 |
| `baselines/` | 基线模型 | 只读 |
| `pretrained_models/` | 预训练模型权重 | 只读 |

- 新文件必须放到正确的目录中，不要在项目根目录随意创建文件
- 不要生成多余的辅助脚本（如检查脚本、清理脚本等）
- 临时调试文件用完后立即删除

### 脚本限制

项目中只允许以下脚本，**禁止新增任何脚本**：

预计算：
- `scripts/1_precompute.sh` — 预计算 shell 脚本
- `scripts/1_precompute.py` — 预计算 Python 主程序

训练：
- `scripts/train.sh` — 一键入口（自动预计算 + 训练）
- `scripts/2_train.sh` — 训练 shell 脚本
- `scripts/2_train.py` — 训练 Python 主程序

如需调整逻辑，应修改现有脚本或配置，而非创建新文件。

### 模型限制

主模型设计（`DTransformer/` 目录下的模型架构）**不可更改**，包括但不限于：

- 模型结构、层定义、前向传播逻辑
- 注意力机制、嵌入方式等核心设计

如需修改模型，必须先与用户确认。
