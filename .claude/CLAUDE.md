## 开发规范

### 必读文件

每次执行新的任务前，**必须**先阅读以下内容，了解当前配置和项目全貌：

1. `README.md` — 项目结构和使用方法
2. `configs/default.yaml` — 所有训练参数、预设模式的唯一配置来源
3. `docs/` 目录下的文档 — 项目设计、训练指南、配置指南等

不得脱离这些文件自行假设参数或结构。

### 文档同步

每次对项目代码或配置进行修改后，**必须**同步更新所有相关文档，包括但不限于：

- `README.md` — 项目结构、使用方法变更
- `docs/` 目录下的对应文档 — 设计细节、参数说明变更
- `scripts/README.md` — 脚本使用说明变更

确保文档与代码始终一致，避免文档过时。

### Git 工作流（强制）

为避免训练中的工作区被破坏，以及保证 `main` 始终可复现，后续开发必须遵循以下 Git 规则：

1. **禁止直接在 `main` 上开发**
   - 默认不允许在 `main` 分支直接修改、提交、推送
   - 所有功能、修复、文档同步、实验性改动都必须先创建任务分支

2. **每个任务对应一个独立分支**
   - 分支命名统一使用：`feat/*`、`fix/*`、`docs/*`、`exp/*`、`release/*`
   - 不要在同一个分支里混合多类改动
   - 任务分支在合并回 `main` 后应删除本地分支；下一个任务重新创建新分支，不复用旧分支
   - 远程任务分支默认保留，只有用户明确要求时才删除，便于回滚和对照历史

3. **训练运行期间禁止在当前工作树切分支**
   - 如果当前目录正在执行训练、预计算或长时间实验，禁止在该工作树中执行 `git switch`、`git checkout`、`git merge`、`git rebase`
   - 需要并行开发时，必须优先使用 `git worktree` 新建独立工作目录

4. **合并前先验证**
   - 改代码后至少完成语法检查或最小链路验证
   - 改训练/预处理流程后至少完成对应数据集的 smoke test 或最小运行验证
   - 改代码或配置后必须同步更新文档

5. **`main` 只保留稳定状态**
   - 只有在任务分支验证通过后，才允许合并回 `main`
   - 合并时优先保留清晰边界，避免把实验中间态直接推进 `main`
   - 合并完成后应清理已完成的本地任务分支；远端任务分支默认保留

6. **版本号只在合并到 `main` 后更新**
   - `v4.x` 版本提交必须对应已验证、已合并到 `main` 的状态
   - 版本说明必须写清楚改进内容和影响范围

### 项目整洁

每次执行完命令后，**必须**检查项目中是否产生了垃圾文件或冗余信息，包括但不限于：

- 临时文件、缓存文件（`__pycache__`、`.pyc` 等）
- 未使用的配置文件或残留文件
- 重复或过时的文档内容

发现后及时清理，保持项目目录干净整洁。

每次执行结束后，必须主动清理本次创建的临时文件、缓存文件和中间产物，只保留用户明确需要的结果文件。

### 文件管理约束

项目目录结构有明确分工，生成或修改文件时**必须**遵循以下规则：

| 目录 | 用途 | 规则 |
|------|------|------|
| `scripts/` | 可执行脚本（预计算、训练） | 禁止新增脚本，只修改现有 |
| `configs/` | 配置文件 | 训练参数统一在 `default.yaml` 管理 |
| `docs/` | 项目文档 | 修改代码后必须同步更新对应文档 |
| `DTransformer/` | 模型代码 | 禁止修改模型架构 |
| `utils/` | 工具函数 | 可新增工具模块 |
| `data/` | 数据文件 | 只读，不要修改或生成数据 |
| `output/` | 训练输出 | 自动生成，不要手动编辑 |
| `logs/` | 训练日志 | 自动生成，不要手动编辑 |
| `baselines/` | 基线模型 | 只读 |
| `pretrained_models/` | 预训练模型权重 | 只读 |

- 新文件必须放到正确的目录中，不要在项目根目录随意创建文件
- 不要生成多余的辅助脚本（如检查脚本、清理脚本等）
- 临时调试文件用完后立即删除

### 脚本限制

项目中只允许以下脚本，**禁止新增任何脚本**：

预计算：
- `scripts/1_precompute.sh` — 预计算 shell 脚本
- `scripts/1_precompute.py` — 预计算 Python 主程序

训练：
- `scripts/train.sh` — 一键入口（自动预计算 + 训练）
- `scripts/2_train.sh` — 训练 shell 脚本
- `scripts/2_train.py` — 训练 Python 主程序

如需调整逻辑，应修改现有脚本或配置，而非创建新文件。

### 模型限制

主模型设计（`DTransformer/` 目录下的模型架构）**不可更改**，包括但不限于：

- 模型结构、层定义、前向传播逻辑
- 注意力机制、嵌入方式等核心设计

如需修改模型，必须先与用户确认。
